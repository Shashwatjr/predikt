# Myprediktion — Loyalty & Points Program (Final)

**Tagline:** *Predict together. Brag forever.*  
Shown on the login / startup boot screen.

## The organizing principle
**Only ONE currency has real value (Aura). Everything else is a score, not money.**

## The three points, one job each
- **AURA** — the only thing worth real value (future: vouchers). Earn: win predictions in real-stakes rooms + gated referral. Doors close to skill + referral only when Aura becomes redeemable.
- **RIZZ** — creator / growth score, never redeemed. Earn: create rooms, bring people, streaks. Rizz climbs Levels.
- **GEMS** — fun-play score, never redeemed. Earn: be right in friends / family rooms. Gems are for bragging rights.

## Levels (climbed with Rizz)
| Lvl | Title |
|-----|-------|
| 1 | **Rising** |
| 2 | **Sharp** |
| 3 | **Prophet** |
| 4 | **Legend** |
| 5 | **G.O.A.T** |

**G.O.A.T** is verification-gated: real creator, 200+ followers, weekly-active. It is not unlocked by points alone.

Thresholds are illustrative and deliberately hard. Starting planning ranges:
- Level 1: ~100 Rizz
- Level 2: ~1,000 Rizz
- Level 3: ~10,000 Rizz
- Level 4: ~50,000 Rizz

These thresholds are tuned with live data later.

## Milestones & badges (Rizz)
- Per-room badges: 5 / 15 / 25 participants in one room
- Lifetime participant bands to start: 100 / 1,000 / 10,000 lifetime participants

Starting badge set:
- **Warm-up**
- **Full House** — 25+ participants in one room
- **Packed Arena** — 50+ participants in one room
- **Consistent Host** — weekly streak
- **Sharpshooter** — win streak

## Strategic mapping
- Aura-wins → skill / retention
- Aura-referral → acquisition
- Rizz → supply / virality
- Gems → habit / frequency

## Intent-driven stance
Per the current product stance: inclusivity now, bucks on creator side later.

### While Aura has NO cash value (now)
Rewards should stay generous and inclusive. We can reward consistent or large Gems owners with:
- Rizz
- status
- faster Level progression
- even Aura-as-a-score

The current goal is bringing people together, not payouts.

### Launch gate — the day Aura becomes redeemable
Close the non-skill Aura doors:
- Gems → Aura
- headcount → Aura

If those doors stay open after redemption starts, voucher budget leaks.

This is a **future switch**, not a pilot restriction. It also depends on account-integrity Tier 1 being complete first.

## Scope
This entire loyalty / points system is **post-MVP**.

It does **not** gate the tunnel test. It is built after the pilot.

The main reason to define it now is so the `reward_tier` contract field is shaped correctly from the start.
